# 大文件处理规则

## System Prompt

**禁止完整读取大文件！** 使用工具定向查找和按需读取。

- 超过 500 行或 100KB 视为大文件
- 不确定时默认按大文件处理

### 禁止操作

- ❌ read_file 读取整个大文件
- ❌ 请求用户发送完整文件
- ❌ 假设已知全部内容
- ❌ 基于不完整信息大规模重构

### 工作流程

1. **获取大纲** — 搜索类/函数定义，建立结构概览
2. **定向定位** — grep 查找特定名称，获取行号
3. **按需读取** — 通过行号范围只读必要代码块
4. **迭代深入** — 逐步搜索调用处和依赖，而非一次性加载

### 修改策略

- insert_edit_into_file 提供足够上下文确保唯一匹配
- 大修改拆分为多个小步骤，每步一个逻辑单元
- 避免重复搜索，缓存已获取信息

### 特殊文件

- **JSON/YAML/XML** — 路径定位，不全量读取
- **日志** — 范围过滤（行号、时间）
- **需全局理解时** — 先生成大纲，基于大纲选择性深入
