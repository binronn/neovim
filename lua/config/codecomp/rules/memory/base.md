# 记忆管理准则

## System Prompt

当涉及到项目内容/文件更新，或首次分析项目时，必须遵守记忆管理准则
使用 `memory` 工具在 `/memories` 目录下读写文件，作为跨会话的长期记忆系统。

## 存储策略

仅在以下情况存储，勿记琐碎对话：
- **架构决策** — 技术栈、设计模式、复杂逻辑
- **任务状态** — 长任务进度、TODO、已解决坑点
- **用户偏好** — 特定编码风格、禁用库、个性化配置
- **环境信息** — 特定路径、别名、工具链配置

## 命名规范

`[类别]_[具体描述].md`，小写+下划线：
- `dotfiles_zsh_aliases.md`
- `project_auth_logic.md`
- `workflow_nvim_plugins.md`

## 文件格式

```markdown
# 记忆标题
- **最后更新**: YYYY-MM-DD
- **标签**: #tag1 #tag2

## 背景
简述核心内容。

## 详细数据/代码
存储的具体逻辑、命令或配置。

## 权限
- **主动调用**：可主动使用 memory 工具
- **无需标记**：涉及过去项目背景时，直接调用 view/read
- **自动读取**：首次获权时，优先读取记忆后再分析问题
- **透明操作**：使用记忆时简要说明原因
- **内容规范**：要在确保写入内容满足语义的前提下，将保存的内容最大化缩减，言简意赅
