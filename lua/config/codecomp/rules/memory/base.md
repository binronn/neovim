# 长期记忆管理准则 (Memory Tool Rules)

你拥有通过 `memory` 工具在 `/memories` 目录下读写文件的权限。这被视为你的长期记忆系统，用于跨会话存储关键信息。

## 1. 存储策略 (When to Store)
不要记录琐碎的对话，仅在满足以下条件时使用 `create` 或 `insert`：
- **架构决策**：记录项目中选定的技术栈、设计模式或复杂的逻辑流程。
- **任务状态**：对于需要跨多次对话完成的长任务，记录当前的进度、待办事项 (TODO) 和已解决的坑。
- **用户偏好**：记录用户明确要求的特定编码风格、禁用的库或个性化配置逻辑。
- **环境信息**：记录特定于用户机器的路径、别名或工具链配置（如 dotfiles 细节）。

## 2. 命名规范 (Naming Convention)
文件命名必须清晰、小写，并使用下划线分隔，以便于检索：
- **格式**：`[类别]_[具体描述].md`
- **示例**：
  - `dotfiles_zsh_aliases.md`
  - `project_auth_logic.md`
  - `workflow_nvim_plugins.md`

## 3. 文件结构规范 (Internal Structure)
每个存储在 `/memories` 目录中的文件应遵循以下格式：
```markdown
# 记忆标题
- **最后更新**: YYYY-MM-DD
- **标签**: #tag1 #tag2

## 背景/描述
简述这段记忆的核心内容。

## 详细数据/代码块
存储的具体逻辑、命令或配置。
